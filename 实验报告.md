# 环境数据信息平台实验报告

## 一、实验名称

基于 Qt 的环境数据信息平台设计与实现

## 二、实验目的

1. 掌握 Qt Widgets 图形界面开发方法，能够使用常见窗体部件与布局管理构建桌面应用。
2. 掌握 Qt 信号与槽机制，实现界面交互、模块通信和事件驱动处理。
3. 掌握 Qt 网络编程基础，能够通过 UDP 方式获取模拟环境传感器数据。
4. 掌握 SQLite 数据库在 Qt 中的集成方式，实现数据持久化存储与查询统计。
5. 掌握多模块桌面系统的设计方法，完成登录、采集、查询、报警、导出和设置等完整业务流程。
6. 在项目实现过程中培养软件工程意识，能够根据需求进行模块划分、结构整理和功能验证。

## 三、实验环境

1. 开发语言：C++17
2. 开发框架：Qt Widgets
3. 构建工具：CMake
4. 数据库：SQLite
5. 图表组件：QCustomPlot（仓库内置源码，可选启用）
6. 操作系统：Windows
7. 主要 Qt 模块：Widgets、Sql、Network、PrintSupport

## 四、项目背景与需求分析

本项目面向环境监测场景，目标是实现一个集用户认证、实时数据采集、历史查询、异常报警、数据导出和系统设置于一体的桌面平台。系统围绕温度、湿度、PM2.5、CO2 四类环境参数展开，通过本地 UDP 模拟服务提供实时数据，再由主程序完成显示、存储、统计与告警。除主程序外，项目还提供了独立的 random_UDP 模拟器，用于课堂演示、联调与离线测试。

根据实验任务书，系统应覆盖以下 6 个核心模块：

1. 登录与用户管理模块
2. 实时数据采集与展示模块
3. 历史数据查询与统计模块
4. 异常报警模块
5. 数据导出模块
6. 系统设置模块

从当前项目代码看，以上 6 个模块均已具备可运行实现，且系统已形成较清晰的分层结构，具备继续扩展 TCP、串口、Excel 导出等能力。

## 五、系统总体设计

### 1. 总体架构

项目采用“主窗口协调 + 页面展示 + 服务层 + 仓储层 + 数据库管理”的结构：

- main.cpp：程序入口，初始化 QApplication、主题、语言和数据库，并负责登录成功后的主窗口拉起。
- MainWindow：系统总协调器，负责页面装配、信号槽连接、数据采集调度、设置加载保存及跨模块联动。
- pages：界面页面层，分别实现实时页、历史页、报警页、导出页、设置页、用户管理页。
- services：业务服务层，实现报警判定、CSV 导出、UDP 采集等通用业务逻辑。
- repositories：数据访问层，分别对应用户、采样数据和报警记录的数据表操作。
- DatabaseManager：数据库连接、建表初始化、默认管理员创建和仓储对象提供者。
- random_UDP：独立 UDP 模拟器，负责监听主程序请求、生成环境样本并返回 JSON 数据。

该结构将界面逻辑、业务逻辑、数据访问逻辑进行了分离，便于维护和扩展。

### 2. 主程序与模拟器协同关系

当前系统采用“双程序联调”方式：

1. random_UDP 作为本地数据源，默认监听 8888 端口。
2. 主程序内部的 UdpSensorClient 周期性发送 GET 数据报请求样本。
3. 模拟器根据用户设置的数值范围生成平滑变化的环境数据。
4. 主程序收到 JSON 数据后更新 UI、写入数据库，并触发后续报警与统计链路。

这种设计相比把模拟数据直接写死在主程序内部更接近真实采集场景，也更利于后续替换成真实硬件设备或网络网关。

### 3. 业务流程

系统运行流程如下：

1. 启动程序，初始化主题、语言、数据库连接。
2. 打开登录对话框，用户可登录、注册或重置密码。
3. 登录成功后进入主界面。
4. 主界面定时向本地 UDP 模拟器发送请求，获取 JSON 格式环境数据。
5. 接收到数据后更新实时页面，并写入数据库。
6. 新数据进入报警服务进行阈值判断，若超阈值则弹出报警框并写入报警表。
7. 用户可在历史页查询指定时间范围数据并查看统计结果。
8. 用户可在导出页将历史数据、统计结果、报警记录导出为 CSV 文件。
9. 用户可在设置页调整刷新频率、主题、语言、报警提示方式并进行数据库维护。
10. 当用户切换语言或执行注销操作时，系统会在同一 QApplication 进程内重建相应窗口，避免整程序重启。

## 六、数据库设计

系统当前使用 SQLite，本地数据库文件在程序 AppDataLocation 下自动创建。初始化时自动建表并创建默认管理员账号。

### 1. 用户表 users

字段说明：

1. id：主键，自增
2. username：用户名，唯一
3. password_hash：密码哈希值
4. role：用户角色，admin 或 user
5. created_at：创建时间戳

### 2. 环境数据表 env_data

字段说明：

1. id：主键，自增
2. ts：采样时间戳
3. temperature：温度
4. humidity：湿度
5. pm25：PM2.5
6. co2：CO2 浓度

并对 ts 建立索引，以提升按时间范围查询的性能。

### 3. 报警表 alarms

字段说明：

1. id：主键，自增
2. ts：报警时间戳
3. param：报警参数名称
4. value：异常值
5. threshold：触发时阈值

同样对 ts 建立索引，以便快速查询报警记录。

### 4. 密码安全设计

当前项目使用 MD5 对密码进行哈希存储，避免明文保存用户密码。该实现满足课程实验的基础安全要求，但若用于实际生产环境，建议升级为带盐哈希算法。

## 七、各模块设计与实现

### 模块 1：登录与用户管理模块

#### 功能实现

1. 用户登录
2. 用户注册
3. 密码重置
4. 管理员默认账号初始化
5. 管理员添加用户、删除用户、切换角色、重置密码
6. 普通用户登录后不显示用户管理页

#### 关键实现说明

1. 登录界面由 LoginDialog 实现，采用 QTabWidget 将登录、注册、找回密码整合在同一对话框中。
2. 使用 QLineEdit 输入用户名和密码，密码框采用 Password 模式隐藏输入。
3. 登录时调用 UserRepository::validateUser 校验用户名和密码哈希值。
4. 注册时默认新用户角色为 user。
5. 数据库首次初始化时自动创建 admin/admin123 默认管理员。
6. MainWindow 根据当前登录用户角色决定是否加载“用户管理”标签页。

#### 技术要点

- Qt 对话框与表单布局
- 信号与槽机制
- SQLite 用户表操作
- MD5 密码加密存储
- 基于角色的简单权限控制

### 模块 2：实时数据采集与展示模块

#### 功能实现

1. 使用 UDP 从本地模拟器获取环境数据
2. 支持 1 秒、5 秒、10 秒刷新频率切换
3. 实时显示温度、湿度、PM2.5、CO2 数值
4. 通过进度条和图表展示数据变化趋势
5. 支持暂停采集和继续采集
6. 支持显示数据源状态和连接异常提示
7. 模拟器支持自定义四项环境参数的最小值、最大值和监听端口

#### 关键实现说明

1. MainWindow 中的定时器按刷新周期触发采集。
2. UdpSensorClient 向 127.0.0.1:8888 发送 GET 数据报。
3. 模拟器 random_UDP 收到请求后，生成平滑随机游走数据并以 JSON 返回。
4. 主程序解析 JSON 后构造 EnvSample，刷新实时页面指标卡片和曲线图。
5. 新样本同时写入 env_data 表，供历史查询与统计使用。
6. 折线图组件 LineChartWidget 默认保留最近 50 条真实采样，并预测未来 5 条趋势数据，以虚线绘制。
7. 若本地 UDP 服务未启动或返回内容格式异常，界面会标记连接异常并在状态栏显示错误信息。

#### 技术要点

- QUdpSocket 网络通信
- QTimer 定时采集
- QWidget、QLabel、QProgressBar 实时显示
- QCustomPlot 绘制多指标趋势图
- JSON 数据解析

### 模块 3：历史数据查询与统计模块

#### 功能实现

1. 按开始时间和结束时间查询历史环境数据
2. 支持对温度、湿度、PM2.5、CO2 显示列进行筛选
3. 计算指定时间范围内各指标最小值、最大值和平均值
4. 以表格方式显示明细与统计结果
5. 对时间范围非法情况进行提示

#### 关键实现说明

1. HistoryPageWidget 提供时间范围选择和参数勾选界面。
2. MainWindow 接收查询请求后，调用 SampleRepository::querySamples 和 queryStats。
3. querySamples 使用时间范围条件查询 env_data 表，返回指定数量记录。
4. queryStats 通过 SQL 聚合函数 MIN、MAX、AVG 统计四类指标。
5. 查询结果填充到 QTableWidget 中，统计结果展示在下方统计表格。

#### 技术要点

- QDateTimeEdit 时间选择
- QTableWidget 数据展示
- SQL 条件查询与聚合统计
- 查询异常处理

### 模块 4：异常报警模块

#### 功能实现

1. 用户可设置四类环境参数阈值
2. 用户可开启或关闭报警功能
3. 用户可设置报警冷却时间，避免同一指标重复高频触发
4. 数据超限时弹出提示框并记录到数据库
5. 支持按时间范围查询报警记录
6. 支持一键清空报警记录

#### 关键实现说明

1. AlarmPageWidget 提供阈值输入、报警间隔配置和报警记录查询界面。
2. AlarmService 专门负责超阈值判定和冷却控制。
3. 当采样值大于阈值时，系统生成 AlarmEvent，并写入 alarms 表。
4. MainWindow::showAlarmEvents 统一负责弹窗提示和声音提醒。
5. 当前声音提醒使用 QApplication::beep()，并支持“提示音/静音”两种模式。

#### 技术要点

- 阈值配置与持久化
- 信号槽触发告警
- 冷却时间控制
- QMessageBox 可视化提示
- 报警记录数据库存储与查询

### 模块 5：数据导出模块

#### 功能实现

1. 导出历史数据
2. 导出统计结果
3. 导出报警记录
4. 支持选择导出路径
5. 历史数据导出支持选择参数字段

#### 关键实现说明

1. ExportPageWidget 负责采集导出时间范围与勾选条件。
2. MainWindow 根据导出类型查询数据库数据。
3. CsvExporter 使用 QFile 和 QTextStream 按 UTF-8 编码写入 CSV。
4. 系统对导出失败、取消路径选择等场景进行了提示处理。

#### 技术要点

- QFileDialog 路径选择
- QFile 与 QTextStream 文件输出
- CSV 数据组织与写出

### 模块 6：系统设置模块

#### 功能实现

1. 刷新频率设置
2. 报警提示方式设置
3. 界面主题切换
4. 中英文语言切换
5. 数据库备份
6. 数据库恢复
7. 历史数据清理
8. 设置自动保存并在下次启动时生效

#### 关键实现说明

1. SettingsPageWidget 提供刷新频率、报警声音、主题、语言等设置项。
2. QSettings 用于持久化保存刷新频率、阈值、声音模式、主题模式和语言设置。
3. ThemeManager 支持跟随系统、浅色、深色以及 skyblue、deepblue、forestgreen、sunsetorange、violetpurple、neutralgray、sakurapink、lemonyellow 等扩展色彩主题。
4. LanguageManager 通过应用内翻译映射实现中英文切换，并在切换后重建窗口。
5. 数据库备份和恢复通过复制 SQLite 数据库文件完成。
6. 历史数据清理通过删除指定天数之前的 env_data 记录实现。

#### 技术要点

- QSettings 设置持久化
- QPalette 主题调色板切换
- 翻译管理与界面重建
- 数据库文件备份恢复

## 十一、存在的问题与改进方向

虽然项目主体功能已完成，但从工程完善度看仍有以下可提升点：

1. 密码加密方式仍为 MD5，安全性一般，后续可升级为带盐 SHA-256 或 bcrypt。
2. 当前仅支持 UDP 采集，可继续增加 TCP 或串口接入，提升真实设备适配能力。
3. 历史统计结果当前只用表格展示，后续可补充柱状图或饼图可视化。
4. 数据导出目前仅支持 CSV，可继续接入 QXlsx 等库实现 Excel 导出。
5. 声音报警当前为系统蜂鸣，后续可替换为自定义音频文件播放。
6. 查询与报警记录尚未分页，数据量增大后可继续优化性能和交互体验。
7. 权限体系目前较简单，仅区分管理员和普通用户，未来可扩展更细粒度的角色控制。
8. 当前主程序中的 UDP 服务地址和端口仍为代码常量，后续可放入设置页做成可配置项，减少联调成本。

## 十二、实验总结

本实验基于 Qt Widgets、SQLite 和 UDP 网络通信，实现了一套完整的环境数据信息平台。系统已能够完成用户登录与管理、实时采集与展示、历史查询与统计、异常报警、数据导出、系统设置等核心功能，形成了较完整的环境监测业务闭环。

在实现过程中，我进一步掌握了 Qt 桌面程序的界面布局、信号槽机制、数据库操作、网络通信和多模块协同设计方法，也认识到一个完整应用不仅需要实现功能，还需要考虑模块解耦、数据持久化、异常处理和后续扩展能力。

总体来看，该项目已较好地达成实验目标，既满足课程要求，也具备继续扩展为更完整监测平台的基础。

### 2. 运行说明

1. 先启动 random_UDP 模拟器。
2. 在模拟器中确认监听端口为 8888，并点击“启动服务”。
3. 再启动主程序 Natural-Monitor-Brain。
4. 使用默认管理员账号登录：admin / admin123。
5. 进入系统后即可查看实时数据、历史记录、报警和导出功能。
